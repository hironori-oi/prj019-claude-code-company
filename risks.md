## 起案時点リスク R-019-01〜R-019-05

### 最優先対処（Phase 0 で必ず確認・判定）

#### R-019-01: Open Claw の正体・API 公開状況が未確認

- **確率**: 高　**影響**: 高　**スコア**: 最重
- **症状**:
  - clawbro.ai が「コンセプトサイト」「クローズドβ」「特定地域限定」「すでにサービス停止」のいずれかである可能性
  - API / SDK が未公開で、サードパーティ統合（claude-code-company との連携）が技術的に不可能
  - 運営者が個人 / 匿名団体 / 信頼性低い場合、「オーナー位置」に据えるのはセキュリティ上不可
- **緩和策**:
  1. **CB-R-01〜03 を Phase 0 最優先タスク** として即時着手
  2. 一次情報（公式 Web）+ 二次情報（Crunchbase / LinkedIn / GitHub / WHOIS）でクロスチェック
  3. 「正体不明」「API 非公開」「運営信頼性低」のいずれか確定 → DEC-019-XXX で **凍結 or 中止** を即時決裁
- **エスカレーション**: CB-R-01 完了直後に CEO 報告

#### R-019-02: 自律エージェントへの過剰権限による損失リスク（**本案件の核心リスク**）

- **確率**: 高（権限境界設計が甘い場合）　**影響**: 高（最悪シナリオで実損発生）　**スコア**: 最重
- **症状**:
  - **金銭損失**: Open Claw が API キーを使って高額な OpenAI / Anthropic / Vercel / インフラ課金を発生させる
  - **コード漏洩**: claude-code-company 配下のソース（PRJ-001〜PRJ-018 全て）が外部に流出する
  - **第三者サービス改変**: Open Claw が GitHub / Vercel / Supabase に対して書込操作を行い、運用中のプロダクト（PRJ-005 カミレス / PRJ-009 おはなしカルテ / PRJ-015 Coatly / PRJ-017 ホメコト 等）を破壊する
  - **個人情報・認証情報の漏洩**: `.env` / `~/.aws/credentials` / `~/.ssh/` / Better Auth セッション 等
- **緩和策**:
  1. **ハーネスエンジニアリング設計（CB-P-01 / CB-S-01）が本案件の最重要 deliverable**
  2. MUST / MUST-NOT 表で権限境界を明示、最低 30 項目以上
  3. ファイルシステム書込は `projects/PRJ-019/app/` 配下のみ等、明示的ホワイトリスト
  4. シェル実行はホワイトリスト方式（`git status` / `npm test` 等のみ、`rm -rf` / `curl POST` は禁止）
  5. 金銭発生操作（API 課金 / インフラ契約 / ドメイン購入）は **人間エスカレーション必須**、Open Claw 単独では実行不可
  6. 即停止スイッチ（kill switch）を Phase 0 で設計、Phase 1 着手前に動作確認
  7. レビュー部門の CB-S-03 で最悪シナリオ 3 つを想定、ハーネスでブロック可能か検証
- **エスカレーション**: ハーネスでブロック不能なシナリオが 1 つでも残る → Phase 1 着手不可、CEO 即時報告
