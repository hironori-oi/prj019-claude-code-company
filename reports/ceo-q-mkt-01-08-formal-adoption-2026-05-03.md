## §2. DEC 4 件の詳細

### §2.1 DEC-019-026: Q-Mkt-02 公開タイミング 6/20 確定

- **決裁内容**: PRJ-019 Phase 1 完了レポート + LP + ブログ + 自社 HP 反映 + 社内ナレッジ K1〜K10 公開を **2026-06-20 (土) 朝** に同時公開
- **理由**:
  - Phase 1 完了 (6/13 金) + 1 週間調整期間で BAN drill #1/#2 結果反映 + 副作用ゼロ最終確認の安定運用が確保できる
  - 前倒し (6/13 即日) は BAN drill 結果未消化リスク、後ろ倒し (6/27 以降) は SEO momentum 喪失
- **代替案却下**: B (6/13 前倒し) / C (6/27 後ろ倒し) / D (Phase 2 後)
- **影響**:
  - Web 運営部門: 6/12 までに LP / ブログ / HP 段落差替を完了させる必要
  - Marketing: 6/14-19 で 6 営業日の最終調整窓
- **連動**: DEC-019-027 (Heading) + DEC-019-028 (部分開示) + DEC-019-029 (HP 配置) + DEC-019-030 (G-Top-1) の 4 件と同時公開対象

### §2.2 DEC-019-027: Q-Mkt-04 Heading A 採用

- **決裁内容**: 公開時メイン Heading は **A 案「AI 組織が AI 組織を運営する」** で確定
- **理由**:
  - 最大訴求力（subject + verb 重複の認知衝撃 + Open Claw 上流 personal AI assistant 再ポジションを内包）
  - Marketing メッセージング比重 40-25-20-15 と整合（harness 工学最前面）
  - PRJ-018/019 並走で他社差別化 4 軸（日本語 UI / おしゃれ / 組織統合 / ローカル永続化）の差別化軸と一貫
- **代替案却下**:
  - B 案「Open Claw が我が社の意思決定を引き継いだ」: 過剰な擬人化リスク + 法務懸念
  - C 案「個人開発 AI 組織のフロンティア」: 訴求力弱（個人開発感が中小企業向け B2B 受託訴求と乖離）
- **影響**: HP トップ Hero / OG image / プレスリリース原稿 / SNS 投稿の全面 A 案準拠

### §2.3 DEC-019-028: Q-Mkt-05 部分開示モード採用

- **決裁内容**: Phase 1 公開時の開示範囲を **部分開示モード** に確定。具体配分:
  - **harness engineering**: 80% 開示（HITL 7 種 / mock-claude / TimeSource / G-Top-1〜4 等は技術ブログで詳細公開、ただし Open Claw 内部実装の一部は伏字）
  - **org 詳細**: 50% 開示（部署構成 / 並列発注フローは公開、各部署のプロンプト詳細は非公開）
  - **cost 数値**: 100% 開示（Anthropic Max $200 + ChatGPT Pro $200 + 月次 ハードキャップ $300 = 計算根拠込みで完全公開、競合との比較性確保）
  - **ToS 詳細**: 概要のみ（OpenAI ToS / Anthropic ToS の根拠条文は伏字、判定方針のみ公開）
- **理由**:
  - 全面開示 (100%) は Open Claw 上流 OSS リポへの上流提案リスク + 競合 (Cursor / Codex) の即模倣リスク
  - 完全クローズは Marketing 訴求力ゼロ
  - 部分開示は Heading A の訴求と整合 + ToS 詳細伏字で法務リスク低減
- **代替案却下**: A (全面 100% 開示) / C (完全クローズ)
- **影響**: 技術ブログ / プレス / HP 事例ページの編集ガイドラインに本配分を反映

### §2.4 DEC-019-029: Q-Mkt-06 HP 配置 + リード導線

- **決裁内容**:
  - **配置**: HP **トップページに訴求バナー** (Hero 直下) + **事例ページに PRJ-019 Clawbridge 詳細記事** の両方を採用
  - **リード導線**: **Contact form のみ** (`mailto:` フォールバック付き、Calendly / 即時対話チャット導入は Phase 2 以降)
- **理由**:
  - トップ訴求のみは事例詳細導線が不足、事例ページのみは認知獲得が不足 → 両方が最適
  - リード導線を Contact form のみに絞る理由 = Phase 1 直後はリード処理キャパが Owner + CEO のみ、Calendly 即時対話は処理能力超過リスク
  - メールベース対応 (CLAUDE.md 「クライアント対応」規約) と整合
- **代替案却下**: トップのみ / 事例のみ / Calendly 直接予約 / live chat
- **影響**: 自社 HP (`projects/COMPANY-WEBSITE/`) 改修タスクが追加。Web 運営部門 W3 中盤 (6/3 頃) 着手予定
